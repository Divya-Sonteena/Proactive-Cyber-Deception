What is a Honeypot?

A honeypot is a fake computer system intentionally exposed to attract attackers. Everything they do is recorded — while they believe they are on a real server.

Two honeypots deployed:
Cowrie — Emulates SSH and Telnet servers
Dionaea — Emulates SMB, HTTP, FTP, MySQL
🪙 Canary Tokens — Fake files/credentials that alert on access
🔤

What is Tokenization?

Tokenization converts raw log events into a compact, standardized vocabulary so the AI models can process them consistently.

SCAN → RECON → LOGIN_ATT → LOGIN_OK → EXEC → FILE_XFER
SCAN — Port/network scan
RECON — Fingerprinting the target service
LOGIN_ATT — Failed login attempt
LOGIN_OK — Successful authentication (danger!)
EXEC — Shell command executed
FILE_XFER — File upload or download
FILE_WRITE — File written to disk
PRIV_ESC — Privilege escalation attempt
TUNNEL — Covert channel or C2 connection
MALWARE — Malicious binary detected
NET_CONN — Network connection established
SESS_END — Session terminated
🤖

Attack Classification (DistilBERT)

DistilBERT is a transformer model trained to read a token sequence and classify it as BENIGN or MALICIOUS with a confidence score.

Input: [COWRIE] SCAN RECON LOGIN_ATT LOGIN_OK EXEC FILE_XFER
Output: MALICIOUS — 94.2% confidence
95.6% accuracy
93.9% precision
96.9% recall
🔮

Anomaly Detection & Next Action (XLNet)

XLNet learns the structure of attack sequences. A low perplexity means the sequence follows a known attack pattern — flagged as anomalous. It also predicts what the attacker will do next.

Sequence: SCAN → RECON → LOGIN_ATT × 4
Predicted next: LOGIN_OK — brute-force may succeed
Perplexity: low → HIGH anomaly score
→ FILE_XFER — Malware download risk
→ EXEC — Remote code execution risk
→ TUNNEL — C2 / lateral movement risk
→ PRIV_ESC — Full compromise risk
⚠

How is Risk Calculated?

The combined severity formula fuses three independent signals into a single risk score:

severity = (token_sev − 1) × 0.25 + attack_prob × 1.50 + anomaly × 1.25
LOW — Routine scanning, no access gained
MEDIUM — Brute-force or limited execution
HIGH — Successful login or privilege escalation
CRITICAL — Active exploitation, malware, or file deletion
🧠

AI Prevention Advice (Groq LLaMA)

For every HIGH/CRITICAL event, the system sends the full attack context to Groq LLaMA to generate structured, context-specific prevention advice. Results are cached in MongoDB for 24 hours.

Categories returned:
🔴 Access Control — Immediate, short-term, long-term
🌐 Network Security — Firewall, IDS, egress rules
🖥 Host Hardening — EDR, patch, filesystem monitoring
Falls back gracefully to static templates when no API key is configured or Groq is unavailable.
